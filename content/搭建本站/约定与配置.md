# 约定与配置

## 内容目录约定（content/）

本站默认从 `content/` 下加载文档内容，并根据文件夹层级生成左侧目录树：

```
content
  - Java
    - 基础入门
      - 基本介绍.md
  - Spring Boot
    - 基础概念
      - 什么是 SpringBoot.md
```

对应的访问路径（slug）规则：

- `content/Java/基础入门/基本介绍.md` -> `/docs/Java/基础入门/基本介绍`
- `content/搭建本站/MDX 能力演示.mdx` -> `/docs/搭建本站/MDX%20能力演示`

## Frontmatter（可选）

每个文档可在顶部使用 Frontmatter 控制标题、排序和描述：

```md
---
title: 我的标题
description: 一句话简介（用于搜索结果摘要/列表描述）
order: 10
---
```

说明：

- `title`：优先级最高；不写则从正文 `# 一级标题` 推断；再不写则取文件名
- `order`：用于排序；数字越小越靠前；如果你希望某个分类始终排在最后，可以给一个很大的 `order`（例如 `99900`）

## 目录排序（_order.json）

如果你希望手动控制某个目录下**同级目录/文件**的排序，可以在该目录创建一个 `_order.json`：

```json
{
  "order": ["简介", "快速搭建", "约定与配置", "MDX 能力演示", "部署到 Linux 服务器"]
}
```

说明：

- 配置文件作用于**当前目录的直接子项**。
- `order` 里可以写「文件名（不含扩展名）」或「目录名」；写成 `xxx.md` / `xxx.mdx` 也可以。
- 未在列表中出现的子项，会继续按现有规则排序（`order` Frontmatter -> 名称）。
- 根目录排序可以放在 `content/_order.json` 中。

## Markdown（.md）能力

`.md` 文档支持：

- GFM：表格、任务列表、删除线等
- 代码块：带复制按钮 + 代码高亮
- Mermaid：使用 ` ```mermaid ` 代码块自动渲染
- 图片：点击后进入图片查看器（支持缩放/旋转/还原/自适应）

示例：Mermaid

```mermaid
flowchart LR
  A[Write] --> B[Build Index]
  B --> C[Search]
```

## MDX（.mdx）能力（React 组件级内容）

`.mdx` 在 Markdown 的基础上，额外支持直接在文档中使用 **React 组件**，从而实现更强的交互与表达能力（例如：交互式示例、可复用的 Callout/提示块、按钮、图表预览器等）。

本站已内置并暴露了一些可在 MDX 中直接使用的组件（示例）：

- `<Callout type="info" title="提示">...</Callout>`
- `<Counter initial={0} />`
- `<Tabs><Tab title="...">...</Tab></Tabs>`
- `<Steps><Step title="...">...</Step></Steps>`
- `<Terminal>...</Terminal>`
- `<FileTree>...</FileTree>`
- `<Expandable title="...">...</Expandable>`
- `<Badge type="beta" />`
- `<Definition term="CSR">...</Definition>`
- `<Kbd>⌘</Kbd>`
- `<KeyPoint title="...">...</KeyPoint>`
- `<QuoteCard author="..." source="...">...</QuoteCard>`
- `<Checklist><CheckItem checked>...</CheckItem></Checklist>`

你可以在下一篇《MDX 能力演示》中看到完整例子。

### 常用组件用法速查

#### Callout：信息提示块

```mdx
<Callout type="warning" title="注意">
  React 19 严格模式下，某些副作用可能会被触发两次（开发环境）。
</Callout>
```

#### Tabs：多版本/多语言对照

````mdx
<Tabs>
  <Tab title="Java">
    ```java
    System.out.println("Hello");
    ```
  </Tab>
  <Tab title="Kotlin">
    ```kotlin
    println("Hello")
    ```
  </Tab>
</Tabs>
````

#### Steps：写安装/部署/迁移教程

```mdx
<Steps>
  <Step title="安装依赖">npm i</Step>
  <Step title="本地启动">npm run dev</Step>
</Steps>
```

#### Terminal：终端块（带复制）

```mdx
<Terminal>
{`npm i
npm run dev`}
</Terminal>
```

#### FileTree：目录结构展示（带复制）

```mdx
<FileTree>
{`content/
  Java/
    基础入门/
      基本介绍.md`}
</FileTree>
```

#### Expandable：可折叠内容

```mdx
<Expandable title="展开查看：更多说明">
  这里可以放一段较长的补充内容或注意事项。
</Expandable>
```

#### KeyPoint：强调关键点（比 Callout 更“轻”）

```mdx
<KeyPoint title="建议">
  给读者一个可直接执行的结论，减少“看完还要自己总结”的成本。
</KeyPoint>
```

#### QuoteCard：更舒服的引用块

```mdx
<QuoteCard author="OpenAI Docs" source="Writing docs">
  文档的价值在于：减少用户做选择的成本。
</QuoteCard>
```

#### Checklist：结构化清单（替代任务列表）

```mdx
<Checklist title="上线前检查">
  <CheckItem checked>目录树自动生成</CheckItem>
  <CheckItem checked>搜索（支持中文）</CheckItem>
  <CheckItem>未来可扩展：更多 MDX 组件</CheckItem>
</Checklist>
```
