# 项目介绍

Refinex Blog 是一个基于 React 的文档型博客系统，目标是提供接近 OpenAI Developers 文档的阅读体验。

## 核心特性

| 特性 | 说明 |
|------|------|
| 内容驱动 | 把 Markdown / MDX 文件放进 `content/` 目录，自动生成目录树、搜索索引和访问路径 |
| 三栏布局 | 左侧目录导航 · 中间文档阅读 · 右侧页面大纲 |
| 站内搜索 | 基于 FlexSearch 的全文检索，原生支持中文分词 |
| MDX 组件 | 在文档中直接使用 React 组件：Tabs、Steps、Callout、Terminal 等 |
| 开发者工具 | 内置 25+ 在线工具：JSON 格式化、正则测试、Diff 对比、白板等 |
| 亮暗主题 | 支持 Light / Dark / System 三种模式，一键切换 |

## 技术栈

- **React 19** + **TypeScript** — 类型安全的 UI 开发
- **Vite 7** — 极速的开发服务器与构建工具
- **Tailwind CSS v4** — 原子化样式方案
- **Radix UI** — 无障碍优先的 UI 基础组件
- **MDX** — 在 Markdown 中编写 React 组件
- **FlexSearch** — 轻量级全文搜索引擎

## 适用场景

- 个人技术博客 / 学习笔记
- 团队内部知识库
- 开源项目文档站点
- 技术分享与教程系列

## 项目结构概览

```
Refinex-Blog/
├── content/              # 文档内容（Markdown / MDX）
├── public/               # 静态资源
├── src/
│   ├── config/           # 站点配置（导航、个人信息、书签）
│   ├── docs/             # 文档系统核心（渲染、索引、搜索、组件）
│   ├── pages/            # 页面组件（首页、文档、工具、关于）
│   ├── providers/        # 主题 Provider
│   └── components/ui/    # 通用 UI 组件
├── vite.config.ts        # Vite 构建配置
└── package.json
```
