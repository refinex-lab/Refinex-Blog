# MDX 组件指南

MDX 是 Markdown 的超集，允许在文档中直接使用 React 组件。当你需要 Tabs 切换、步骤引导、提示框等交互能力时，将文件扩展名改为 `.mdx` 即可。

<Callout type="info" title="无需手动导入">
  本站已将所有内置组件注册到全局，在 `.mdx` 文件中可以直接使用，无需 `import` 语句。
</Callout>

## Callout 提示框

用于突出重要信息。支持 6 种类型：`info`、`note`、`tip`、`success`、`warning`、`danger`。

```mdx
<Callout type="tip" title="推荐做法">
  将复杂的配置拆分为多个小文件，便于维护和复用。
</Callout>
```

<Callout type="tip" title="推荐做法">
  将复杂的配置拆分为多个小文件，便于维护和复用。
</Callout>

<Callout type="warning" title="注意">
  修改 `vite.config.ts` 后需要重启开发服务器才能生效。
</Callout>

<Callout type="danger" title="危险操作">
  `rm -rf` 命令不可撤销，请务必确认目标路径。
</Callout>

## Tabs 标签页

适合展示多语言代码对照、多方案对比等场景。

````mdx
<Tabs>
  <Tab title="npm">
    ```bash
    npm install
    ```
  </Tab>
  <Tab title="yarn">
    ```bash
    yarn
    ```
  </Tab>
  <Tab title="pnpm">
    ```bash
    pnpm install
    ```
  </Tab>
</Tabs>
````

<Tabs>
  <Tab title="Java">

            System.out.println("Hello, World");
        }
    }

  </Tab>

    ```kotlin

    }
    ```
  </Tab>
  <Tab title="TypeScript">
    ```typescript

    ```
  </Tab>
</Tabs>

## Steps 步骤引导

## FileTree 目录树

可视化展示项目结构，自带复制按钮。目录以 `/` 结尾，文件不带。缩进使用两个空格表示层级。

```mdx
<FileTree title="项目结构">
{`src/
  config/
    site.ts
    profile.ts
  docs/
    contentIndex.ts
    MarkdownRenderer.tsx
    MDXRenderer.tsx
  pages/
    home/
    docs/
    tools/`}
</FileTree>
```

## Expandable 折叠区域
将补充说明、可选阅读等内容折叠起来，减少页面干扰。

```mdx
<Expandable title="为什么选择 Vite 而不是 Webpack？">
  Vite 基于原生 ES Module，开发环境启动速度极快，且内置了对 TypeScript、JSX、CSS Modules 的支持，
  配置更简洁。对于文档型站点来说，Vite 是更合适的选择。
</Expandable>
```

<Expandable title="为什么选择 Vite 而不是 Webpack？">
  Vite 基于原生 ES Module，开发环境启动速度极快，且内置了对 TypeScript、JSX、CSS Modules 的支持，
  配置更简洁。对于文档型站点来说，Vite 是更合适的选择。
</Expandable>

</KeyPoint>

## 组件组合使用

这些组件可以自由嵌套组合。例如在 Steps 中使用 Terminal，在 Callout 中使用行内代码：

<Steps>
  <Step title="创建 MDX 文件">
    将文件扩展名设为 `.mdx`，例如 `05_MDX 组件指南.mdx`。
  </Step>
  <Step title="使用组件">
    直接在文档中编写组件标签，无需导入。
  </Step>
  <Step title="预览效果">
    保存文件后开发服务器会自动刷新，即可看到渲染结果。
  </Step>
</Steps>
