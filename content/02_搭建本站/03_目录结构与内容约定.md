# 目录结构与内容约定

本站采用「文件即路由」的设计：`content/` 下的文件夹层级直接映射为左侧目录树，`.md` / `.mdx` 文件即页面内容。

## 命名约定

目录和文件统一使用 `XX_实际名称` 格式命名：

- `XX_` 是两位数字前缀，用于控制排序（数字越小越靠前）
- 渲染时前缀会被自动去除，只显示实际名称
- URL 路径中同样不包含前缀

```
content/
  01_开始使用/
    01_快速开始.md
  02_搭建本站/
    01_项目介绍.md
    02_快速开始.md
    03_目录结构与内容约定.md
  03_Spring 生态/
    01_Spring Framework/
    02_Spring Boot/
```

上面的结构会生成如下 URL：

| 文件路径 | 访问地址 |
|---------|---------|
| `content/01_开始使用/01_快速开始.md` | `/docs/开始使用/快速开始` |
| `content/02_搭建本站/03_目录结构与内容约定.md` | `/docs/搭建本站/目录结构与内容约定` |

## 新增内容的步骤

1. 在 `content/` 下创建目录或文件，按 `XX_名称` 格式命名
2. 编写 Markdown 或 MDX 内容
3. 保存后开发服务器自动刷新，目录树即时更新

如果需要调整顺序，只需修改文件名的数字前缀即可，无需编辑任何配置文件。

## Frontmatter

每篇文档可以在顶部添加 YAML 格式的 Frontmatter：

```markdown
---
title: 文档标题
description: 一句话描述，用于搜索结果和列表展示
cover: https://example.com/cover.png
author: 作者名
createdAt: 2026-01-01
updatedAt: 2026-02-01
---
```

各字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 否 | 文档标题。不写则从正文第一个 `# 标题` 推断，再不写则取文件名 |
| `description` | 否 | 简短描述，出现在搜索结果中 |
| `cover` | 否 | 封面图 URL，显示在文档顶部 |
| `author` | 否 | 作者名称 |
| `createdAt` | 否 | 创建日期，格式 `YYYY-MM-DD` |
| `updatedAt` | 否 | 更新日期，格式 `YYYY-MM-DD` |

> 排序完全由文件名的 `XX_` 前缀控制，Frontmatter 中无需写 `order` 字段。

## .md 与 .mdx 的选择

| 格式 | 适用场景 |
|------|---------|
| `.md` | 纯文本内容：教程、笔记、API 文档等 |
| `.mdx` | 需要交互组件的内容：可切换的代码示例、步骤引导、可折叠区域等 |

两种格式可以在同一目录下混用，渲染效果一致。区别在于 `.mdx` 支持直接使用 React 组件。
